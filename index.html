1.Classes

What is a class?

  • A class is like a template or blueprint that defines:
    • Attributes(data/state)
    • Behaviors(methods/functions)

Why use classes?

  • To create structured, modular and reusable code.
  • Helps in modeling real-world objects (like Person, Car, BankAccount).

Snytax example

    
  public class Car {
    String model;
    int year;

    void startEngine() {
        System.out.println("The " + model + " engine has started.");
    }
  }

2.Object

What is an object?

  • Is an Instance of a class.
  • Created using the new keyword.
  • Has its own values for the class's fields.

How to create an object

    
  Car car1 = new Car();
  car1.model = "Toyota Corolla";
  car1.year = 2020;
  car1.startEngine(); // Output: The Toyota Corolla engine has started.

Real-life Analogy

  • Class = recipe
  • Object = cake made from the recipe

3.Arrays

what is an array?

  • A fixed-size collection of elements of the same datatype.
  • Index-based(starts at zero).
  • Must specify size when created.

Syntax:

    
  int[] numbers = new int[5];         // Empty array of size 5
  String[] names = {"Anna", "Ben"};   // Initialized array

Accessing and Modifying:

    
  numbers[0] = 10;
  System.out.println(numbers[0]); // 10

Iterating:

    
  for (int i = 0; i < numbers.length; i++) {
    System.out.println(numbers[i]);
  }

Limitation:

  • Size is fixed once defined.
  • Not flexible for dynamic data.

4.List (Interface)

What is a List?

  • Is an interface in Java.
  • Represent an ordered, resizable collection.
  • Can hold duplicate elements.
  • The most common implementation is an Arraylist

Example:

    
  List<String> fruits = new ArrayList<>();
  fruits.add("Apple");
  fruits.add("Banana");
  System.out.println(fruits.get(1)); // Banana

5.Arraylist

What is an Arraylist?

  • A resizable array implementation of the List interface.
  • Useful when data size is unknown or dynamic.

Features:

  • Fast access via index
  • Allow duplicates
  • Can grow/Shrink dynamically

Example:

    
  ArrayList<Integer> numbers = new ArrayList<>();
  numbers.add(10);
  numbers.add(20);
  numbers.remove(0); // Removes 10

6.Methods

What is a Method?

  • A block of code that performs a specific task
  • defined once and reused.
  • Can return a value or be void.

Example:

    
  int multiply(int a, int b) {
    return a * b;
  }

Overloading:

Same name, different parameters.

    
  void greet() { System.out.println("Hi"); }
  void greet(String name) { System.out.println("Hi, " + name); }

7.Abstraction

What is Abstraction?

  • Hiding internal details, showing only what's necessary.
  • Simplifies usage and reduces complexity.

Two ways in Java:

  • Abstact class
  • Interface

Example:

    
  abstract class Shape {
    abstract double area();
  }

  class Circle extends Shape {
    double radius;

    Circle(double r) { radius = r; }

    double area() {
      return Math.PI * radius * radius;
    }
  }

8.Inheritance

What is Inherince?

  • Hiding internal details, showing only what's necessary
  • Simplies usage and reduces complexity.

Syntax:

    
  class Animal {
    void eat() {
        System.out.println("Eating...");
    }
  }

  class Dog extends Animal {
      void bark() {
          System.out.println("Barking...");
      }
  }

Example:

    
  Dog d = new Dog();
  d.eat(); // Inherited
  d.bark(); // Own method

10. Encapsulation

What is Encapsulation?

  • Keeping data safe by hiding it inside a class.
  • Fileds are private, accessed via public methods.

Example:

    
  class Account {
    private double balance;

    public void deposit(double amount) {
      if (amount > 0) balance += amount;
    }

    public double getBalance() {
      return balance;
    }
  }
